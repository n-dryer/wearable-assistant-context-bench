# Probe-Study Archive

This document preserves the older probe-study framing that predated the
current public benchmark wording. It is retained for historical
reference only and should not be used as the source of truth for the
current benchmark.

## Archived terminology

Earlier internal drafts described the project as a probe study focused
on `reference-state selection`, a `four-policy` rubric, and related
terms such as `policy cell`, `golden set`, `open coding`, and
`stale-context drift`.

Those terms are historical. The current benchmark language is simpler:

- use `benchmark` instead of `probe study`
- use `earlier context` and `current context` instead of `reference state`
- use `judge label` instead of `policy`
- use `prompt conditions` instead of `intervention conditions`

## Why this archive exists

The older wording appears in past notes, early implementation history,
and previous internal planning material. Keeping a short archive helps
explain those references without letting outdated language leak into
the active repo surface.
