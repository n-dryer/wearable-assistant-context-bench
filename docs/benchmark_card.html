Wearable Assistant Context Benchmark

A product benchmark for comparing models on cross-turn reference resolution under context change in a wearable multimodal assistant.

101 frozen scenarios 3 prompt conditions 2 trials per cell balanced Turn 2 accuracy

Product problem

Users should not have to keep restating what they are looking at, holding, or referring to after they move, switch objects, change screens, or otherwise change context.

Canonical examples:

  • Bedroom to kitchen. The user asks about the bedroom wall, walks into the kitchen, and asks about the wall again. The assistant should answer about the kitchen, not the bedroom.
  • Hammer to screwdriver. The user asks about a hammer, swaps to a screwdriver, then asks "how do I use this?" The assistant should answer about the screwdriver.

What v1 measures

Canonical v1 measures whether the assistant answers about the right thing after the user's context changes.

Measured task Cross-turn reference resolution under context change.
Primary score Balanced Turn 2 accuracy over the current and prior scored classes under baseline.
Secondary metric Simulated repair rate, used as a proxy for likely user correction cost after a miss.

Scope notes

  • Built from real user feedback and product testing.
  • Spoken user questions are part of the cue set, but canonical v1 uses transcript proxies rather than raw audio.
  • Canonical v1 does not yet directly test raw acoustic grounding, speaker attribution, or ambient audio cues.
  • The benchmark is narrow on purpose and optimized for model-selection usefulness.
