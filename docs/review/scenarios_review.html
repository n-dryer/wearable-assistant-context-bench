Wearable Assistant Context Benchmark v1

Scenario Review Sheet

50 scenarios. One per page. Read all four channels (camera + user speech, Turn 1 and Turn 2), then mark Pass, Edit, or Replace. Add notes if needed.

What's in this bank

Total scenarios50
Correct frame to useabstain (2), clarify (3), current (33), prior (12)
Difficulty mixeasy (15), hard (15), medium (20)
Shift typesObject swap (in hand) (12); Object state change (8); Sequential task step (6); Location change (6); Attention shift (same scene) (5); Absent referent (5); Screen content change (4); Recall from before Turn 1 (4)

Before you mark Pass, confirm

  1. Camera identifies the object. Read the Turn 1 and Turn 2 camera descriptions cold, without seeing the answer vocabulary. You should be able to identify what's in frame from the description alone. If you can't, the description is too vague.
  2. The user speech sounds natural. Would a real person actually say that out loud to a wearable? No narration of what's visible. No announcement that something changed.
  3. The shift is hidden in the camera. Turn 2 user speech does not give away that the context changed. The model has to notice the difference by integrating the camera and the question.
  4. The shift fits the type. The shift type at the top of the page tells you what kind of shift this is supposed to be. Make sure the scenario actually does that.
  5. Answer vocabulary is reasonable. The judge-only vocabulary at the bottom should cover the right things for both contexts (object name, technique, state). You don't need to memorize it. Just sanity-check it makes sense.
  6. Repair line is fair. If the model misses Turn 2, the repair line names the intended frame explicitly. Read it and confirm it gives a model a fair second shot.
